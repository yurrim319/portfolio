LUT 보간 — 왜 R/G 기준값이 같아야 하는가

구버전 vs 신버전 코드의 차이를 그림으로 이해하기

전제 — 지금 보간하려는 씬 컬러

씬 컬러가 R=0.9, G=0.1, B=0.3 이라고 가정합니다.

r15 = 0.9 × 15 = 13.5 → floor=13, ceil=14, frac=0.5 g15 = 0.1 × 15 = 1.5 → floor=1, ceil=2, frac=0.5 b15 = 0.3 × 15 = 4.5 → floor=4, ceil=5, frac=0.5

실제로 LUT에서 찾아야 할 위치는 슬라이스4와 슬라이스5 사이의 R=13.5, G=1.5 지점입니다.
이 지점을 정확히 보간하려면 그 주변 4개 코너 픽셀이 모두 필요해요.

정확한 Bilinear — 4개 코너가 필요한 이유

아래 격자에서 ★ 위치(R=13.5, G=1.5)를 구하려면 주변 4점이 있어야 해요.

1
R=13.5 기준으로 하단 두 점 보간
lerp(좌하, 우하, 0.5) → 하단 중간값
2
R=13.5 기준으로 상단 두 점 보간
lerp(좌상, 우상, 0.5) → 상단 중간값
3
G=1.5 기준으로 위 두 결과 보간
lerp(하단중간, 상단중간, 0.5) → ★

R과 G가 독립적으로 각자의 축에서 보간돼요. 4점이 목표를 정확히 둘러싸고 있어요.

구버전의 문제 — 대각선 2점

구버전은 R과 G를 동시에 floor/ceil 했어요. 그러면 샘플 포인트가 어떻게 되는지 보세요.

구버전 코드
U_floor = B_off + floor(r15) ← R=13 V_floor = floor(g15) ← G=1 U_ceil = B_off + ceil(r15) ← R=14 V_ceil = ceil(g15) ← G=2 R과 G가 동시에 올라감

S_floor = (R=13, G=1) 코너

S_ceil = (R=14, G=2) 코너

→ 대각선 방향 양 끝점만 샘플

R=0.9, G=0.1처럼 R과 G가 비대칭인 경우엔 대각선 lerp가 완전히 빗나가요.

R=0.9(거의 오른쪽), G=0.1(거의 아래쪽)이면 목표는 오른쪽 아래인데,
구버전은 R 비율(0.9)을 대각선에 적용해서 왼쪽 아래→오른쪽 위 대각선의 90% 지점을 결과로 내요. 완전히 빗나가죠.

신버전 — R/G는 GPU에게, B만 수동

신버전 코드
U = B_off + r15 ← 소수점 그대로 13.5 V = g15 / 15.0 ← 소수점 그대로 0.1 R/G는 소수점 UV → GPU Bilinear 슬라이스4에서 샘플 (4점 자동 처리) 슬라이스5에서 샘플 (4점 자동 처리) result = lerp(S_floor, S_ceil, fb) ← B만 수동으로 섞음

S_floor = 슬라이스4에서 GPU가 4점 보간한 결과

S_ceil = 슬라이스5에서 GPU가 4점 보간한 결과

→ R/G 위치가 동일한 두 점만 비교
→ B 비율 fb로 선형 보간

두 슬라이스에서 같은 R/G 좌표로 샘플한 결과를 B 비율로 섞어요.
R/G 보간은 GPU가 각 슬라이스 안에서 올바르게 처리하고, B 보간만 셰이더가 담당해요.

Q5 — 채널을 다른 비율로 재조합하면 안 되는 이유

S_floor는 이미 LUT가 처리한 완성된 출력 색상이에요. 채널을 쪼개서 다른 비율로 섞으면 원래 LUT 의도가 깨져요.

구버전 — 채널마다 다른 비율
S_floor = LUT[R=13, G=1, B=4]의 출력 = (0.82, 0.44, 0.10) ← 하나의 색상 S_ceil = LUT[R=14, G=2, B=5]의 출력 = (0.95, 0.51, 0.33) ← 하나의 색상 result.r = lerp(0.82, 0.95, fr=0.9) ← R 비율 result.g = lerp(0.44, 0.51, fg=0.1) ← G 비율 (다름!) result.b = lerp(0.10, 0.33, fb=0.5) ← B 비율 (다름!)
신버전 — 하나의 비율
S_floor = 슬라이스4, R=13.5, G=1.5 의 출력 = (0.88, 0.47, 0.12) ← GPU 보간 완료 S_ceil = 슬라이스5, R=13.5, G=1.5 의 출력 = (0.91, 0.49, 0.18) ← GPU 보간 완료 두 점의 R/G 위치가 동일 result = lerp(S_floor, S_ceil, fb=0.5) 모든 채널을 같은 비율로 섞음

핵심 차이

구버전: 서로 다른 3D 위치의 두 색상을 채널마다 다른 비율로 분해해서 재조합 → LUT가 의도한 색이 아닌 임의의 색

신버전: 같은 R/G 위치의 두 색상(B만 다름)을 B 비율 하나로 통째로 섞음 → LUT 의도 그대로 보간
