01 RGB 3D Cube 구조

LUT의 본질은 3D 큐브예요. 씬 컬러 하나(R, G, B)가 큐브 안의 한 점이고, LUT는 그 점을 변환된 색상으로 바꿔주는 테이블입니다. 큐브를 돌려보면서 각 꼭짓점 색상이 RGB 조합임을 확인해보세요.

드래그로 회전
R 축 — X방향
Red 채널
오른쪽으로 갈수록 R↑
슬라이스 내 U(가로) 좌표에 대응
G 축 — Y방향
Green 채널
위로 갈수록 G↑
슬라이스 내 V(세로) 좌표에 대응
B 축 — Z방향
Blue 채널
앞뒤 방향 = 슬라이스 번호
R/G와 달리 UV 좌표가 아님 → 수동 보간 필요
R축 (→)
G축 (↑)
B축 (앞↔뒤)
각 점의 색상 = 해당 위치의 RGB 값
02 B축 슬라이싱 → 256×16 스트립

GPU에서 3D 큐브를 직접 쓰는 대신, UE5는 큐브를 B축 기준으로 16장으로 잘라 가로로 이어붙입니다. 아래 뷰어에서 애니메이션이 자동으로 슬라이스를 분리/합체해요. B가 슬라이스 번호라는 게 뭔지, R/G가 각 슬라이스 내 UV라는 게 뭔지 확인하세요.

드래그로 회전 · 슬라이스 자동 분리/합체
슬라이스 구조
B값 → 슬라이스 번호
B=0 → 슬라이스 0 (가장 뒤)
B=0.5 → 슬라이스 7~8 사이
B=1 → 슬라이스 15 (가장 앞)

각 슬라이스는 R×G 16×16 격자
왜 B만 수동 보간인가
슬라이스 사이 = 물리적 공백
슬라이스는 텍스쳐 상에서 가로로 떨어진 별개 영역이에요.
GPU Bilinear는 하나의 슬라이스 안에서만 자동 보간해요.

슬라이스 경계를 넘는 보간은 직접 두 슬라이스를 샘플해서 lerp(S0, S1, fb)로 처리해야 해요.
낮은 B값 슬라이스 (어두운 파랑)
높은 B값 슬라이스 (밝은 파랑)
B 보간 연결선 (슬라이스 사이)
03 구버전 vs 신버전 샘플링 비교

씬 컬러 R=0.9, G=0.1, B=0.3을 LUT에서 찾는 경우예요. 버튼으로 구버전/신버전 샘플 포인트를 켜고 끄면서 ★ 목표 위치와 실제 샘플 위치의 차이를 비교하세요. 목표는 오른쪽 아래 코너에 있어요. 구버전이 왜 틀렸는지 보여요.

드래그로 회전 · 아래 버튼으로 켜고 끄기
표시:
구버전 — 빨간 점 2개
대각선 샘플의 문제
floor(R,G)와 ceil(R,G)를 동시에 적용하면
샘플이 (R_floor, G_floor)와 (R_ceil, G_ceil)이에요.

R=0.9(오른쪽 끝), G=0.1(아래쪽 끝)이면
★목표는 오른쪽 아래인데 구버전 대각선은
왼쪽아래→오른쪽위 방향 → 완전히 빗나감
신버전 — 초록 점 4개
4점 Bilinear + B 수동 lerp
R, G 소수점 그대로 UV에 넘기면
GPU가 4개 코너를 목표 주변에 배치해요.

R_floor/ceil × G_floor/ceil = 4점이 목표를 정확히 둘러쌈
B축만 수동 lerp(S_floor, S_ceil, fb)로 처리
★ 목표 위치 (R=0.9, G=0.1)
구버전 샘플 2점
구버전 대각선
신버전 샘플 4점
신버전 4점 사각형
B 보간 연결선
